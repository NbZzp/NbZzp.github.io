react,vue,Angular对比1.0

发表于 2017-12-01

虽然前端技术发展更新换代非常的迅速（号称3个月更新一次）。但近两年前端框架逐渐形成了vue，react，Angular三分天下的局面。

简介

vue（国内尤大申编写） react（faceBook项目）轻量级视图层框架，不需要操纵dom，改为控制数据。而angular（google）是mvc框架，2.0之前的版本通常我们称之为angularJS，默认的双向绑定简化了代码，但其采脏检测机制，影响了程序的性能。2.0之后的版本之后的叫angular，改善了AngularJS中的格构各种各样的问题，推出了angular-cli脚手架工具，方便构建项目，提供了服务器端渲染（SSR）的支持；用Angular不仅可以写webapp，还可以开发移动端App。

生命周期

Angular：

ngOnchanges 当绑定的输入属性的值发生变化时调用，首次调用一定会发生在ngOnInit
ngOnInit 在第一轮ngOnChanges完成之后调用。此时所有的属性都已经有了正确的初始绑定值
ngDoCheck 在每个Angular变更检测周期中调用
ngAfterContentinit 当吧内容投影进组件之后调用
ngAfterContentChecked 每一次完成被投影的组件内容变成检测之后调用
ngAfterViewInit 初始化完组件视图及其子视图之后调用

ngAfterViewChecked 每次做完组件视图和自视图的变更检测之后调用
ngOnDestroy 当angular每次销毁指令/组件之前调用

react：

当一个组件初始化创建一瞬间：constructor（初始化只执行一次）
当组件将要被挂在到页面上：componentWillMount（一次）
render（渲染，很多次）
组件已经被挂在到页面：componentDidMount（一次）（发ajax请求的地方）
组件将要被销毁的时候：componentWillUnMount（一次）
但数据发生改变的时候：shoulecomponentUpdate
数据即将更新页面的时候：componentWillUpdate
组件完成更新时：componentDidUpdate
父组件从新render的时候，子组件会执行componentWillReceiveProps
